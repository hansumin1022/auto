# ISO 25010 에 의거한 SW 개발 요구사항

### 개발 요구사항

|특성|내용|
|--|----|
|기능적합성|- 1초 주기로 HC-SR04 거리 측정 후 7cm 미만 시 드론 감지로 판단한다<br>- 드론 감지 시 LED 점등, MOSFET CH1·CH2로 금속판을 18~22°C로 유지한다<br>- 드론 감지 시 MOSFET CH3으로 무선충전 패드에 전원을 공급한다<br>- 1초 주기로 INA219를 통해 배터리 전압·전류·잔량을 측정한다<br>- 1초 주기로 DS18B20 #2를 통해 외부 기온을 측정하고 저장한다|
|성능효율성|- OsTask_LandingSwitch는 1초(±10ms) 주기로 실행되어야 한다<br>- 드론 감지 후 MOSFET 제어까지 응답시간은 1초 이내여야 한다<br>- 이벤트 미수신 시 WaitEvent()로 대기하여 CPU를 점유하지 않아야 한다|
|호환성|- ERIKA3(OSEK/VDX) 위에서 다른 Task와 충돌 없이 동작해야 한다<br>- AUTOSAR Classic 레이어 구조를 준수해야 한다|
|사용성|- 드론 감지 여부는 LED로 육안 확인이 가능해야 한다<br>- 감지 거리·목표 온도·배터리 용량은 상수 수정만으로 변경 가능해야 한다|
|신뢰성|- 드론 미감지 시 모든 MOSFET은 즉시 OFF 상태로 전환되어야 한다<br>- 온도센서 읽기 실패 시 MOSFET을 즉시 OFF하여 과열을 방지해야 한다|
|보안성|- RTE 버퍼는 Rte_Write()·Rte_Read() API를 통해서만 접근 가능해야 한다<br>- 배터리 잔량 계산값은 0~100% 범위를 벗어나지 않아야 한다|
|유지보수성|- SW는 Application·RTE·OS 레이어로 명확히 분리되어야 한다<br>- SWC 간 직접 통신은 금지되며 반드시 RTE를 통해 데이터를 교환해야 한다 <br>- 모든 HW 의존 코드는 스텁으로 대체 가능해야 한다|
|이식성|- SIMULATION_MODE 플래그 전환만으로 Ubuntu VM과 TC277 양쪽에서 동작 가능해야 한다<br>- MCU 변경 시 HAL 레이어만 교체하면 상위 SWC 코드는 수정 없이 동작해야 한다|
